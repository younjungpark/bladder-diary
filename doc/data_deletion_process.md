# 사용자 데이터 삭제 요청 운영 절차

이 문서는 `BladderDiary` 사용자로부터 계정 삭제 또는 데이터 삭제 요청을 받았을 때의 운영 절차 초안입니다.

## 1. 목적

- 사용자 데이터 삭제 요청에 일관되게 대응합니다.
- 개인정보 처리방침과 실제 운영 절차를 맞춥니다.
- Google Play 제출 시 데이터 삭제 관련 문의에 대비합니다.

- `BladderDiary 데이터 삭제 요청`

## 3. 삭제 요청 종류

### 1) 계정 삭제

- 로그인 계정 자체를 삭제하고, 서버에 동기화된 사용자 기록도 함께 삭제하는 요청

### 2) 데이터 삭제

- 특정 기록 또는 동기화 데이터 삭제를 요청하는 경우
- 현재는 운영 단순화를 위해 `계정 삭제 요청 = 서버 데이터 전체 삭제 요청`으로 처리하는 것을 기본 정책으로 둡니다.

## 4. 기본 처리 정책

- 요청자 본인 확인 전에는 삭제를 진행하지 않습니다.
- 계정 삭제 요청 시 서버에 저장된 사용자 관련 기록 삭제를 우선 처리합니다.
- 사용자의 기기 내부 로컬 데이터 삭제는 운영자가 원격으로 직접 수행할 수 없습니다.
- 로컬 데이터 삭제가 필요할 경우 사용자에게 아래를 안내합니다.
  - 앱 내 기록 직접 삭제
  - 앱 로그아웃
  - 앱 삭제 또는 앱 데이터 삭제

## 5. 본인 확인 절차

삭제 요청을 받으면 아래 항목을 요청합니다.

- 앱에 가입 또는 로그인할 때 사용한 이메일 주소
- 사용 중인 로그인 방식
  - Google
  - Kakao
  - 이메일 로그인
- 요청 내용
  - 계정 전체 삭제
  - 특정 데이터 삭제 문의

운영자는 아래 기준으로 본인 확인을 진행합니다.

- 요청 이메일과 가입 이메일이 일치하는지 확인
- 필요한 경우 추가 확인 메일 1회를 보냅니다.

## 6. 처리 절차

1. 삭제 요청 메일 접수
2. 본인 확인 메일 회신
3. 확인 완료 후 삭제 범위 확정
4. 서버 저장 데이터 삭제
5. 처리 완료 메일 발송
6. 내부 운영 기록에 처리 일시와 요청 유형만 간단히 남김

## 7. 권장 응답 시간

- 처리 완료 목표: 영업일 기준 7일 이내
